🧭 意图路由设计原则

路由是客服中台的核心决策节点。本设计将问题分为 4 类，通过 LLM 语义分类器进行路由（而非关键词匹配），分类置信度低时默认走 FAQ 链兜底。核心原则：确定化决策交给规则，模糊判断交给 LLM。

📖
FAQ 咨询
1 次 LLM
典型问法：政策是什么、多少天、支持哪些支付…
Router→ FAQ Agent→ 输出回复
🚚
物流查询
2 次 LLM + 工具
典型问法：快递到哪了、订单发货了吗、几天到…
Router→ 物流查询🎭→ 回复生成
🔄
退换货申请
2 次 LLM
典型问法：我要退货、质量有问题、能换货吗…
Router→ 规则核查→ 方案生成
😤
投诉 / 维权
2–3 次 LLM + HITL
典型问法：太差了、要投诉、赔偿、曝光、维权…
Router→ 情绪识别→ 升级判断→ HITL/安抚

Router System Prompt 设计要点

Router 被严格约束只输出 4 个枚举值之一（faq / logistics / return / complaint），禁止解释和加标点。这是「确定化」的典型应用——当分类规则明确时，约束 LLM 输出格式，以保证下游解析的稳健性。

🤖 Agent 设计原则

每个 Agent 职责单一、输入/输出明确、相互解耦。Router 只做分类，不做回复；回复 Agent 只做措辞，不做决策。这使每个节点可独立测试、替换、优化，也便于溯源问题。

⚙️ Human-in-the-Loop 触发设计

HITL 是 AI 客服中台的核心安全机制。不是所有问题都应由 AI 全自动处理——高风险场景必须设计「暂停点」，让人工在关键决策前介入确认。

触发条件优先级处理方式SLA
情绪分 ≥ 7（愤怒/激烈）🔴 高人工实时介入90秒内响应
涉及法律/曝光/媒体关键词🔴 高升级客服主管立即
退款金额 > 500元🟠 中人工审批后执行5分钟
超时未解决（>10min）🟠 中自动转人工自动触发
用户主动要求人工🟡 低正常排队3分钟
AI 置信度 < 0.6🟡 低人工兜底5分钟

指标目标值监控维度说明

🏗 中台治理能力矩阵

  • 可观测性：每次 LLM 调用记录 token 消耗、延迟、输入/输出摘要，支持按场景 Drill-down
  • 灰度发布：新 Agent Prompt 版本先对 5% 流量生效，A/B 对比 CSAT 后全量
  • 降级策略：LLM 超时 > 3s 时走规则引擎兜底，保证 SLA
  • 成本控制：FAQ 链优先走小模型（haiku），复杂链路才调大模型（sonnet）
  • 安全审查：输出经过敏感词过滤 + 品牌口吻校验后再发送给用户

🔄 数据飞轮：Bad Case 驱动持续优化

AI 客服中台的核心竞争力不是上线时的准确率，而是持续迭代的能力。数据飞轮将每次失败案例转化为下一版本的训练信号，形成正向循环。

Bad Case 类型来源信号优先级处理方式
意图误分类人工接入后标注纠偏🔴 P0扩充 Router 训练集 / 优化分类 Prompt

📈 PM 视角：飞轮加速的关键

  • 标注效率：设计清晰的标注界面，单条 Bad Case 标注时间 < 30秒
  • 标注一致性：制定标注规范文档，定期校准（IAA ≥ 0.85）
  • 自动化扩充：用 LLM 生成语义相似变体，人工抽样验证 10%
  • 效果归因：每次 Prompt 迭代记录变更内容，便于回溯和对比
