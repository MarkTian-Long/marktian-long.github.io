# Blog Design — 博客视觉设计规范

本文件定义「思考碎片」博客在文章页和列表页的视觉与交互规范，是 WRITING_GUIDE.md 的补充。新文章参照本规范执行；存量文章不强制回改。

---

## 一、文章页：正文段落节奏

**问题：** 正文 `<p>` 间距默认 `margin-bottom: 16px`，连续多段密度高，长文容易视觉疲劳。

**规范：**
- 新文章的 `<style>` 中将 `.post-body p` 的 `margin-bottom` 设为 `20px`
- 每隔 3-4 个普通段落，主动用 `<h2>` 开新节——避免无结构的长段堆叠
- 不要为了"填满"一节而在同一 h2 下堆超过 6 段正文

```css
/* 新文章 <style> 中覆盖默认值 */
.post-body p { margin-bottom: 20px; }
```

---

## 二、文章页：强调文本克制使用

**问题：** `<strong>` 出现频率过高时，所有重点等于没有重点。

**规范：**
- 每段最多 1 处 `<strong>`
- `<strong>` 只用于**核心判断句**，不用于修饰性词语、产品名、或引用来源
- 同一段内禁止连续出现 2 个以上 `<strong>`

**判断方法：** 把所有加粗文字单独列出来读，如果能形成文章的骨架提纲，则使用合理；如果只是随机强调，则过度。

---

## 三、文章页：decision-card 组件规范

**问题：** 连续 3 个 decision-card 视觉等权重，缺乏呼吸感。

**规范：**
- decision-card 之间 `margin` 用 `24px`（模板默认 `20px` 稍紧，新文章覆盖）
- decision-card 内正文 `<p>` 字号 `13px`，颜色 `var(--text-2)`，不要在卡片内再用 `<strong>`
- 文章若有 3 个以上 decision-card，必须用 `<h2>` 节标题将它们包在同一节下，不要在不同节里各散放 1 个

```css
/* 新文章覆盖 */
.decision-card { margin: 24px 0; }
```

---

## 四、文章页：callout 块内容密度

**规范：**
- callout 内正文不超过 3 句话；超过时将多余内容移回正文节，callout 只留核心结论
- 新文章写完后，在深色主题下截图检查一次 callout 背景对比度（clay-soft 在深色下较淡）
- callout 内禁止嵌套列表或子标题，保持单一文字块

---

## 五、文章页：参考资料节

**规范：**
- 参考资料链接数量建议 ≤ 12 条；超过时按文章节做内部分组

```html
<!-- 超过 12 条时用注释分组 -->
<div class="refs">
  <h2>参考资料</h2>
  <ul>
    <!-- 产品发布相关 -->
    <li>...</li>
    <!-- 竞争格局相关 -->
    <li>...</li>
  </ul>
</div>
```

## 六、文章页：h2 密度与 TOC

**规范：**

| h2 数量 | 处理方式 |
|---------|---------|
| ≤ 2 个 | 不加 TOC，单栏布局 |
| 3-6 个 | 标准双栏布局 + 左侧 toc-card |
| ≥ 7 个 | 考虑拆成两篇，或将部分 h2 降为加粗段落 |

- 每个 h2 节下的正文建议 200-500 字（约 1-4 段）
- 避免一个 h2 下只有 1 段（太短，不值得单独成节），或超过 8 段（太长，应再分节）

## 七、列表页：年月分组

**现状（问题）：** JS 按年份分组（取 `date.split('.')[0]`），13 篇文章全堆在"2026"一组，年份分组失去时间线导航价值。

**规范（目标状态）：** 按 `YYYY.MM` 分组，`year-label` 显示"2026 · 04"。

**实现（`index.html` JS `render()` 函数）：**

```js
// 改为按 date 全字段分组（YYYY.MM）
var byYM = {};
list.forEach(function (p) {
    var ym = p.date; // 直接用 "2026.04"
    if (!byYM[ym]) byYM[ym] = [];
    byYM[ym].push(p);
});
// year-label 显示格式："2026 · 04"
var label = ym.replace('.', ' · ');
```

## 八、列表页：摘要展示方式

**结论：摘要保持静态展示（始终可见），不做 hover 折叠。**

hover 折叠动效在实际使用中体验较差：鼠标滑过列表时内容持续跳动，反而影响阅读效率。摘要的价值在于帮助快速判断是否值得点进去，静态展示更直接。

如文章数量增多（> 30 篇）导致列表过长，优先考虑分页方案（见 WRITING_GUIDE.md"未来考虑"节），而非折叠摘要。

---

## 九、列表页：标签二级过滤

**现状（问题）：** 侧栏仅有 技术/产品/商业 3 个大分类，同类文章增多后粒度不足。

**规范（目标状态）：** 在分类按钮下方动态生成 tag chip 列表，支持和 category 叠加过滤。

**实现要点：**
- `load()` 后统计所有 `tags`（含 `topics`），按出现频次排序，展示为胶囊按钮组
- 新增 `activeTag = ''` 状态变量，`filtered()` 函数追加 tag 过滤逻辑
- tag chip 样式复用现有 `.post-tag`（`--clay-soft` 背景，`--clay` 文字）
- active 状态加 `border: 1px solid var(--clay)`

---

## 十、列表页：搜索与分类状态独立

**现状（问题）：** 搜索词和分类切换之间没有视觉反馈，用户不清楚当前同时生效了哪些过滤条件。

**规范（目标状态）：** 两者状态独立维护（已是），但在搜索框下方展示当前生效的过滤条件 chip。

**实现要点：**
- 搜索框和分类按钮互不清空对方（已实现，维持现状）
- 在 `.search-box` 下方追加 `#activeFilters` 容器，当 `cat !== 'all'` 或 `query` 非空时显示对应 chip
- chip 带 `×` 按钮，点击清除对应条件并重渲染
