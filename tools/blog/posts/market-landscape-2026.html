2026.04 市场格局

2026 年 AI Agent 市场格局：开放 vs 封闭的博弈

Anthropic 的策略转变揭示了平台战的底层逻辑：谁控制了 Harness 层，谁就控制了 Agent 生态的价值分配。

核心判断

2026 年的 AI Agent 市场正在经历一次格局重组。框架大战已经基本结束，赢家已经确定；更重要的转变是：基础设施提供商（Anthropic、OpenAI）开始主动划清界线，把第三方生态的机会成本算清楚了。

框架大战：赢家已确定

2023-2024 年的框架爆炸期已经大幅收窄。目前形成了分工明确的格局：

LangGraph
有状态、可控的工作流
图形化执行模型，支持暂停、等待人工审批、从检查点恢复，适合生产规模的复杂业务逻辑。
AutoGen / 微软 Agent
多 Agent 协作
微软在 2025 年 10 月合并 AutoGen 和 Semantic Kernel，正式押注企业多 Agent 协作场景。
CrewAI
易用性与功能的平衡
基于角色的抽象，非 ML 工程师也能理解和部署。但 6-12 个月后多数团队会遇到能力天花板需要迁移。
LlamaIndex
检索优先的 Agent
从编排框架演进为检索层专家，现在通常搭配 LangGraph/AutoGen 使用，负责知识流水线和 RAG 层。

关键事件：Anthropic 的策略转变

2026 年 4 月 · OpenClaw 禁用案例
Anthropic 阻止第三方 Agent 工具使用订阅配额
从 2026 年 4 月 4 日起，Anthropic 阻止了 Claude Pro/Max 订阅用户通过订阅来支撑 OpenClaw 等第三方 Agent 工具。OpenClaw 当时有 135,000+ 个运行实例，约 60% 的活跃会话通过订阅配额运行。

技术根源：OpenClaw 这类第三方 Harness 框架绕过了提示词缓存优化，一个重度 OpenClaw 会话消耗的基础设施资源远多于等效的 Claude Code 会话——相当于以固定订阅价格获得了无限计算资源。

对用户的实际影响：从固定价格订阅切换为按令牌计费，成本可能增加 50 倍。

这不是偶然，是系统性转变

Anthropic 不是在"消灭竞争对手"，而是在做经济现实检查：订阅模式无法支撑 24 小时的 Agent 重度使用，OpenClaw 无意中成为了 Anthropic 最大的补贴接收者。

Google 同期也对使用第三方工具的 Gemini CLI 用户采取了类似行动。这是所有基础设施提供商都在进行的系统性策略，而非 Anthropic 的个例。

底层逻辑是：当 Agent 工作负载的真实成本变得可见，最初通过开放生态快速增长的 AI 公司，开始把这部分成本算进产品定价里。

未来的可能格局

市场正在形成两层结构：

  • 第一方生态（Anthropic：Claude Code、Claude Projects）：享受完整的基础设施优化和订阅定价
  • API 直连用户（企业直接调用 API）：按量计费，有完整的能力访问
  • 第三方框架：必须采用按量计费或自建基础设施，无法依赖平台补贴

这意味着：构建在第三方 Harness 框架上的 Agent 产品，其经济模型需要重新测算；而能控制 Harness 层的平台方，掌握了 Agent 生态的价值分配权。

对 AI PM 的启示：做 Agent 产品选型时，要把"基础设施提供商的策略方向"算进来。如果你的产品依赖某个第三方框架，要问：这个框架和底层模型提供商的关系是什么？有没有被"断粮"的风险？选择和平台方直接对接，虽然工程成本更高，但依赖关系更清晰。
