# 颜色系统说明

## 本项目的色彩策略

**双轨强调色设计**：区分"行动性元素"和"思想性内容"使用不同颜色，避免单一强调色过多导致稀释。

| 角色 | 颜色 | HEX | 使用场景 |
|------|------|-----|---------|
| 行动色 | 蓝色 | `#2563eb` | CTA 按钮、外链、可点击操作 |
| 思想色 | 陶土 | `#c96442` | Section 标签、观点条目边框、分类标记 |
| 主背景 | 羊皮纸 | `#f5f4ed` | 页面底色 |
| 次背景 | 奶油 | `#f0eee6` | 交替区块（about/contact/tools） |
| 卡片面 | 象牙 | `#faf9f5` | 卡片/弹窗表面 |

## 60-30-10 规则

- **60%** 背景中性色（`--bg-primary` / `--bg-secondary`）
- **30%** 文字、边框、次要元素（`--text-secondary` / `--border`）
- **10%** 强调色（蓝色 CTA + 陶土标签）

强调色的力量来自**稀缺性**，不要到处使用。

## Alpha 系列使用规则

Alpha 变量用于叠加效果（hover 背景、焦点 ring、边框高亮）：

```css
/* 蓝色行动性元素 */
--accent-blue-08    /* hover 背景（极淡） */
--accent-blue-15    /* focus ring / 徽章背景 */
--accent-blue-30    /* 边框 hover 状态 */

/* 陶土思想性元素 */
--accent-clay-06    /* 标签背景 */
--accent-clay-25    /* 标签边框 */
```

**注意**：大量使用 alpha/rgba 是不完整调色板的信号。仅在交互状态、叠加层使用，不要用它做主要背景色。

## 禁忌

- 禁止硬编码颜色：`color: #2563eb` → 改为 `color: var(--accent-blue)`
- 禁止纯黑 `#000` / 纯白 `#fff` 做大面积背景
- 禁止灰色文字叠加在彩色背景上（会显得发灰死板），改用背景色的深色变体
- 不要用 `cyan + dark 背景 + 紫蓝渐变` 的 AI 审美套路

## 新项目适配建议

1. 保留双轨色结构，替换具体色值为你的品牌色
2. 用 OKLCH 定义颜色，而不是 HEX/HSL（感知均匀）：
   ```css
   --accent-primary: oklch(50% 0.18 250);   /* 主色 */
   --bg-base: oklch(96% 0.01 85);           /* 轻度暖色调背景 */
   ```
3. 中性色一定要向品牌色微微偏色（chroma 0.005-0.015），制造潜意识连贯感

## 主题（亮色/暗色）判断标准

不要默认选亮色"安全"或暗色"酷"。根据用户场景决定：

| 适合亮色 | 适合暗色 |
|---------|---------|
| 日间使用、阅读类、轻松场景 | 专业工具、夜间使用、数据密集 |
| 婚礼/生活类应用 | 交易系统、监控面板、创意工具 |
| 医疗/教育类 | 音乐/视频/暗环境 |

本项目选亮色（羊皮纸系）：定位个人品牌站，白天阅读场景为主。
