## 背景

为 ASCI（Artificial Science Intelligence）AI 产品经理面试准备的可交互 Demo。面试官会在面试前独立体验并打分，需要能自行操作、理解产品设计意图。

## 核心决策

| 决策项 | 结论 |
|--------|------|
| 数据模式 | 纯 Mock，不接 API |
| 实现范围 | 先做第一条路径：文献综述（正常完成） |
| 使用场景 | 面试官独立体验，非现场讲解 |
| 推进模式 | 混合：主步骤间手动点击，步骤内子步骤+日志自动播放 |
| 布局方案 | 三列同屏（220px + 1fr + 280px） |
| Coming Soon | 灰色占位卡片，传递完整设计意图 |
| Screen 3 操作 | "接受"（toast 反馈）+ "重新开始"（回 Screen 1） |

## Screen 2：执行过程（核心）

### 三列布局

**左列 220px — 任务拆解树**
- 5 个主步骤纵向排列，可展开子步骤
- 节点信息：图标 + 名称 + 风险标签（绿/黄/红）+ 状态圆点
- Human Checkpoint 节点标 👤
- 状态流转：pending（灰）→ running（蓝脉冲）→ done（绿）
- 当前步骤自动展开，其他折叠

**中列 1fr — 执行日志流**
- 终端风格深黑背景，monospace 字体
- 顶部固定标题 "执行日志 · Agent Runtime"
- 日志逐条 fadeIn，自动滚底
- 级别：INFO（蓝）/ WARN（黄）/ ERROR（红），带时间戳 [HH:MM:SS]
- Checkpoint 时底部弹出内联确认卡片

**右列 280px — 状态面板**
- 置信度圆环（CSS conic-gradient）
- 进度条 步骤 X/5
- 当前步骤信息卡：名称 + 风险 + 调用工具
- 完成时绿色闪烁反馈

### 底部控制栏
- 左：「▶ 执行下一步」按钮
- 右：步骤进度文字
- 执行中按钮变灰 "执行中..."
- Checkpoint 时按钮变 "✅ 确认并继续"

### 交互流程
```
点击"执行下一步"
  → 按钮灰化
  → 左列节点 running（蓝脉冲）
  → 中列日志逐条滚动（300-800ms 随机间隔）
  → 右列置信度/进度更新
  → 子步骤完成 → 节点 done
  → 按钮恢复

Step 3 特殊：完成后按钮变"✅ 确认并继续" + Checkpoint 卡片
```

## Screen 3：结果评估

**区块 1 — 结果内容**
- 综述标题 + 摘要（200 字）+ 3 条关键发现

**区块 2 — 来源溯源**
- 表格：论文标题 / 作者 / 年份 / 期刊 / 相关性评分（6 条）
- 统计行："共引用 6 篇文献，87% 来自 SCI Q1 期刊"

**区块 3 — 可信度评估**
- 三维度条形图：来源质量 8.5 / 推理链路 7.8 / 数据一致性 7.2
- 每条附说明文字
- 综合评分 + 建议（"建议对全文精读步骤的矛盾标注进行人工复核"）

**操作按钮**
- 「✅ 接受结果」→ toast "结果已保存至知识库"
- 「🔄 重新开始」→ 回 Screen 1

## Mock 数据设计

### 任务树（5 步）

```
1. 🔑 关键词提取（低风险）→ 主题词拆解 + MeSH 映射
2. 🔍 数据库检索（低风险）→ PubMed + Semantic Scholar + 去重
3. 📋 摘要筛选（中风险）→ 相关性打分 + 阈值过滤 + 👤 Human Checkpoint
4. 📖 全文精读（高风险）→ 方法论提取 + 关键发现 + 矛盾检测
5. ✍️ 综述生成（中风险）→ 大纲 + 段落撰写 + 引用插入
```

### 日志序列
~35 条日志，每个子步骤 2-4 条。step-4 含 WARN（矛盾检测）。

### 置信度变化
step1: 95% → step2: 88% → step3: 82% → step4: 75% → step5: 78%

### 结果数据
6 条文献来源，三维度评分，3 条关键发现。

## 视觉风格

- 深色主题，CSS 变量前缀 `--asci-*`，自包含
- 背景 #080c18 / 表面 #0d1224 / 强调蓝 #4f8fff
- 绿/黄/红 = 低/中/高风险 = done/warn/error
- 日志面板 #0a0e1a（更深，终端感）
- 动画克制：fadeIn（日志）、pulse（running）、闪烁（完成反馈）

## 技术约束

- 单文件 `tools/asci/index.html`，预估 ~1150 行
- 超 300 行门槛，用 `gen_index.js` 生成后删除
- 面试题覆盖：Q1（风险标注）、Q3（可信度评分）、Q4（Human Checkpoint）
