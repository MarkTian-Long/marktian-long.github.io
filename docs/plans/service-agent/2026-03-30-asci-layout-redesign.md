# ASCI 布局与交互重设计

## Context
当前三列布局（步骤树 / 日志 / 置信度面板）信息割裂，科研内容（论文、关键词、矛盾卡片）被淹没在系统日志里，右列置信度数字对科研工作者无实用意义。用户确认参照 Claude 界面模式重构：左窄（时间线）+ 中宽（主内容）+ 右窄（Artifacts 积累）。

---

## 改动一览（6项）

### A. 左列：步骤树 → 步骤时间线（有内容摘要）

**现状**：只显示步骤名 + 风险标签（低/中/高），无实质信息。

**改后**：
- 移除风险标签（低/中/高）
- 每步完成后在步骤名下方显示关键结果摘要：
  - Step 1 关键词：`8 个检索词`
  - Step 2 检索：`276 篇候选`
  - Step 3 筛选：`21 篇纳入 · 3 篇待定`
  - Step 4 精读：`47 条 Findings · 1 处矛盾`
  - Step 5 生成：`2400 字综述`
- 步骤状态用更有意义的标签替代"低/中/高风险"：
  - `待执行` / `执行中` / `需审核` / `已完成`
- "需审核"步骤用橙色高亮（当前用风险色做的事）

**涉及函数**：`renderTree()` + `.tree-step-*` CSS + `.tree-step-summary` 新增

---

### B. 中列：双区布局（上：步骤主内容 / 下：日志折叠）

**现状**：日志占满中列，步骤结果卡片（`step-result-card`）混插在日志流中，科研内容被日志噪声淹没。

**改后**：
- 中列分为两个区：
  - **上区（主内容区，占 ~65%）**：显示当前步骤的实质内容
    - 执行前：空状态提示"等待执行"
    - 执行完成后：直接展示 `step-result-card` 的内容（关键词/论文列表/矛盾卡片/大纲）
    - 历史步骤内容可通过左列时间线点击切换查看
  - **下区（日志区，占 ~35%，可折叠）**：系统日志，默认展开但高度固定，不再主导视野
    - 顶部有折叠按钮"▲ 收起日志 / ▼ 展开日志"
    - WARN 级别日志自动显示 badge 计数：`⚠ 2 条警告`

**Step 2（数据库检索）补充**：
- 搜索结果进度条下方加"论文预览"折叠块（前5条，标题+年份），可点击打开 paperModal
- 加"导出检索结果"按钮（mock toast："已导出 276 条 BibTeX"）

**Step 3（摘要筛选）补充**：
- "待定"按钮改名为"暂缓（需全文确认）"
- 顶部加决策计数 badge：`✅ 纳入 X 篇  ⏸ 暂缓 X 篇  ❌ 排除 X 篇`
- 阈值滑动条实时联动纳入数：拖动时显示预估变化

**Step 4（全文精读）补充**：
- 矛盾卡片下方加"全部 Findings"折叠块（前5条，每条带出处论文名）
- 加"导出 Findings"按钮（mock toast）

**涉及函数**：`renderStepResult()` 从 `logBody` 改为写入新的 `mainContent` 容器；`appendLog()` 保持写入日志区

---

### C. 右列：置信度面板 → Artifacts 积累面板

**现状**：只显示当前置信度环形图 + 进度条 + 当前步骤工具名，不随时间积累。

**改后**（参照 Claude Artifacts 风格）：
- **置信度曲线**：不再是单个环形图，改为折线迷你图，每步完成后追加一个点（95→88→82→75→78），让趋势可见
- **决策记录流水**：每次 HITL 决策后追加一条（已有 `appendHitlDecisionLog` 逻辑，改为渲染到右列）
- **工具调用记录**：每步用了哪些 Tool，以 tag 形式积累展示
- 移除"当前步骤"卡片中的"低/中/高风险"标签
- 保留"需要你做什么"状态提示（`hitl-status`），但放在右列顶部，更突出

**涉及函数**：`updatePanel()` / `updateHitlStatus()` / `appendHitlDecisionLog()` 输出目标改为右列新结构；`restoreStatusPanelStructure()` 同步更新

---

### D. 布局尺寸调整

**现状**：`grid-template-columns: 220px 1fr 280px`

**改后**：`grid-template-columns: 200px 1fr 240px`
- 左列略窄（时间线不需要太宽）
- 右列略窄（Artifacts 摘要性内容）
- 中列得到更多空间

---

### E. 降级面板移至中列

**现状**：降级时 `statusPanel`（右列）被整体替换为降级选项，导致置信度等消失。

**改后**：降级选项改到中列主内容区展示（降级时中列切换为降级卡片），右列保持 Artifacts 积累不被清空。

**涉及函数**：`showDegradePanel()` 目标改为中列主内容区

---

### F. 其他小修复
- Step 1 启动时左列立即渲染（现在需要点执行才出现）
- 底部控制栏"⚡ 演示降级流程"按钮改为更低调的样式（当前太显眼）

---

## 关键文件
- `tools/asci/index.html`（唯一文件，CSS + HTML + JS 全在里面）

## 不改动范围
- Screen 1（任务选择页）：不动
- Screen 3（结果页）：不动
- MOCK 数据：不改内容，只补充 Step 2/4 的 findings/preview 数据字段
- 底部控制栏（上一步/下一步按钮）：位置逻辑不变

---

## 实现顺序
1. 改 HTML 结构（中列拆两区，右列新结构）
2. 改 CSS（布局比例、新增 `.main-content`、`.log-section`、`.artifacts-*`、`.tree-summary` 等）
3. 改 JS（`renderStepResult` 目标、`renderTree` 加摘要、`updatePanel` 改 Artifacts、`showDegradePanel` 改目标）
4. 补充 Step 2/4 的内容（论文预览、findings 列表）
5. 改"待定"文案 + 决策计数 badge

## 验证
- 浏览器直接打开 `tools/asci/index.html`
- 走完全部 5 步，检查每步主内容区显示正确
- 点击左列历史步骤，检查主内容区切换正常
- 触发降级流程（演示按钮），检查中列正确展示降级卡片，右列 Artifacts 不被清空
- 检查日志折叠/展开
