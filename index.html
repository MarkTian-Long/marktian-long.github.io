🔮 我的预判

2026 年底前，以全自动 Agent 为代表的产品不会在企业端大规模落地。C 端需求复杂度撑不起 Agent 的使用门槛；B 端的真正阻碍不是技术，而是工作流整合、权责归属和内部系统对接——这些问题没有纯技术解法。

写于 2026.03 Agent 落地

国内大模型的过度迎合问题短期内难以系统性改善。根本原因在于训练目标和用户反馈机制的设计取向，在需要高质量批判性输出的工作场景，用户仍需主动设计 prompt 来规避这一问题。

写于 2026.03 模型对齐

金融行业 LLM 落地的主战场在 2027 年前将停留在内部知识库和客服辅助，不会进入决策流程。监管合规要求的可解释性、数据不出域、责任归属明确这三条，目前没有哪个 LLM 产品能同时满足。

写于 2026.03 金融行业

🏭 行业落地

客服 / 售前 规模化落地

标准化问答场景已跑通，头部电商、银行外呼已大规模上线

软件开发辅助 规模化落地

Copilot 类工具已成标配，代码补全、Review 辅助是最快被接受的场景

金融 试点阶段

内部知识库和客服辅助有进展，但合规审批、数据安全要求让决策类应用难以越过监管红线

教育 试点阶段

个性化辅导、作业批改有场景，但如何处理"学生直接用 AI 完成作业"的边界问题还没有答案

医疗 早期探索

辅助诊断、病历整理有探索，但医疗责任归属不明确，AI 输出需要医生全量复核

法律 早期探索

合同审查、案例检索有价值，但律师对准确性要求极高，幻觉问题直接影响可用性

⚠️ 当前能力短板

过度迎合

模型倾向认同用户判断，即使用户有误也不主动纠正。在需要批判性反馈的工作场景（方案评审、需求验证）中可用性低。真正解决意味着：模型能主动识别并指出用户假设中的错误。

客服质检方案评审需求分析
Agent 落地的成本/价值错位

C 端用户日常任务链条不够长，Agent 能力远超实际需求；B 端有复杂任务，但嵌入已有工作流面临内部知识接入、人机协同边界、权责划分等系统性问题。技术不是瓶颈，工作流整合才是。

企业自动化个人助理RPA 替代
长期记忆与上下文持久化

跨会话记忆依赖浅层存储，无真正的语义检索，用户在不同设备、不同会话中无法延续上下文。

个人助理企业知识库长周期辅导
实时感知与行动能力

训练数据有截止日期，实时数据接入依赖外部工具调用，稳定性和一致性不足。
