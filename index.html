💡 对 AI 落地的判断 2026.03 · 持续更新

💬 我的观点

全自动无监督的 Agent 在 2027 年前不会大规模落地
2026.03 Agent 落地 ▾

B 端：真正的阻碍从来不是技术——工作流整合、权责归属、内部系统对接，这三道坎没有纯技术解法，也不会因为模型更强而自动消失。
C 端：平台接入解决的是"用起来"，解决不了"用得稳"。Agent 能产生实际效果的操作（误删文件、代发消息、读取通讯录和聊天记录）往往不可逆或存在隐私风险，一旦出错用户没有补救空间，信任壁垒不会因为入口变了就消失。

此外，当前阶段普通用户的日常需求复杂度本就有限，长链条 Agent 的能力溢出于实际需要——但随着使用习惯的建立，C 端对复杂任务的需求也会逐步释放。

📝 思考碎片：Agent 三大工程问题 →
金融 AI 的近期形态是"有人工节点的半自动化"，而非完全自主 Agent 链
2026.03 金融行业 ▾

B 端：监管与审计要求每一步决策可追溯、可签字，部分节点的法律责任必须由人工承担，推理步骤不透明的 Agent 进不了审批流程。
C 端：大模型不具备直接替代持牌顾问做出最终投资建议的资质，AI 只能辅助决策，不能绕过人工审核环节。

现实的切入点有三个：
· 智能投顾：行业痛点不在涨跌预测，而在用户行为管理——AI 的切入点是情绪陪伴、行为分析和量化逻辑筛选组合，最终建议仍由顾问确认
· 研报生成：投研人员耗时在信息聚合而非分析本身——AI 承担资讯聚合、数据填充、格式校验，分析师专注判断与审核
· 客服路由：大量重复咨询消耗人工资源——AI 识别意图后精准分发，人工专注处理投诉、异议等高复杂度案例

国内大模型与 Claude 在对齐取向上存在实质差异，影响产品设计
2026.03 模型对齐 ▾

Claude 的不同体现在两处：一是批判性输出更克制，不倾向于给用户想听的答案；二是执行任务前会先问清楚"你想要什么结果、有哪些限制"，而不是直接开始——理解意图比快速给输出更重要，国内模型普遍跳过这个环节。

这个差异对 AI PM 定义产品的影响是实质性的：如果你设计的产品以"快速给答案"为核心交互，那么意图澄清机制天然被挤掉；但在企业场景中，一个需求理解错了的快速输出，往往比慢一步确认代价更大。如何在"响应速度"和"意图对齐"之间找到平衡，是当前 AI 产品设计中真正值得深入的问题之一。

AI 技术的投入价值取决于它是"补救缺陷"还是"编码知识"
2026.03 产品框架 ▾

补救缺陷型技术（CoT 提示、角色扮演设定、JSON 格式要求）会随着基础模型能力提升而消亡，因为模型升级后这些缺陷自然消失。编码知识型技术（金融合规规则结构化、多源数据冲突优先级定义、行业约束条件代码化）不会消亡，因为模型不可能从训练数据中推断出特定组织的内部规则。

实际意义：在消亡度高的技术上大量投入，ROI 是递减的；真正值得建立的护城河是领域知识的结构化能力，以及让系统在生产环境可靠运行的工程设计。

📝 思考碎片：技术消亡度框架 →
AI 工程竞争重心已从 Prompt 转向 Harness，2026 年护城河在系统可靠性
2026.04 工程演进 ▾

Prompt Engineering → Context Engineering → Harness Engineering，关注的问题依次从"这次调用输出更好"，到"整个任务的信息怎么组织"，再到"整个 Agent 系统怎么在生产环境里稳定运行"。

三层的本质区别：Prompt Engineering 补救模型缺陷，是过渡技术；Context Engineering 解决信息组织问题，属于编码知识型，持续有效；Harness Engineering 解决系统可靠性问题，与模型能力正交——模型越强、真正暴露出的系统设计问题反而越多。

2026 年的竞争重心已从"谁的 Prompt 写得好"转向"谁的 Harness 设计得更稳"：工具调用的安全边界、失败恢复机制、人工介入节点设计、系统可观测性，这些才是难以复制的工程壁垒。

金融 试点阶段

内部知识库和客服辅助有进展，但合规审批、数据安全要求让决策类应用难以越过监管红线

教育 试点阶段

辅助诊断、病历整理有探索，但医疗责任归属不明确，AI 输出需要医生全量复核

法律 早期探索

合同审查、案例检索有价值，但律师对准确性要求极高，幻觉问题直接影响可用性

⚠️ 当前能力短板 展开 ↓

过度迎合

模型倾向认同用户判断，即使用户有误也不主动纠正。在需要批判性反馈的工作场景（方案评审、需求验证）中可用性低。真正解决意味着：模型能主动识别并指出用户假设中的错误。

客服质检方案评审需求分析
长期记忆与上下文持久化

跨会话记忆依赖浅层存储，无真正的语义检索，用户在不同设备、不同会话中无法延续上下文。

个人助理企业知识库长周期辅导
知识边界的自我认知不足

模型在训练数据覆盖范围内会表现出过度自信，即使面对可能已过时的信息也不主动提示不确定性，需要用户显式要求联网才会验证。真正解决意味着：模型能主动识别"这条信息可能已变化"并自发触发验证，而不是等用户质疑。
