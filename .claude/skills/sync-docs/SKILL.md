# 文档同步

变更内容：$ARGUMENTS（描述刚刚做了什么改动）

## 执行步骤

### 第一步：梳理变更内容
如果用户没有在 $ARGUMENTS 中说明，询问：
- 新增了什么文件或功能？
- 删除或重命名了什么？
- 修改了什么重要配置或规范？

### 第二步：检查需要更新的文档

**始终检查 CLAUDE.md**：
- 目录结构描述是否还准确？
- 已有工具模块表格是否需要新增行？
- 常见修改位置是否需要更新？

**检查 README.md**：
- 项目结构图是否与实际一致？
- 功能模块表格是否需要更新？
- 快速开始部分是否还有效？

**如果新增了工具，检查该工具的 README.md**：
- 是否存在？内容是否完整？
- 功能描述、数据来源、文件结构、维护指南四项是否都有？

**如果新增或修改了博客文章/博客功能，检查博客相关文档**：
- `tools/blog/data/posts-meta.json`：新文章是否已追加条目（slug/date/title/summary/tags/category/url 齐全）？
- `tools/blog/WRITING_GUIDE.md`：写作规范是否需要更新（新增标签、callout 格式变化等）？
- `CONVENTIONS.md` 第七章「博客内容规范」：字段规范或分类原则是否需要更新？

**如果修改了命名规范、目录规范等，检查 CONVENTIONS.md**：
- 对应章节是否需要更新示例或规则？

### 第三步：执行更新
只更新实际有出入的部分，不要重写整个文档。
每处修改说明原因：「因为新增了 xxx 工具，所以在工具表格中添加一行」

### 第四步：确认输出
列出修改了哪些文档的哪些部分，让用户确认。

## 注意事项
- 文档描述要简洁，不要把代码细节写进去（那是代码注释的工作）
- CLAUDE.md 的目录结构和工具表格是最重要的，优先保持准确
- 如果文档和代码有冲突，以代码为准，更新文档
